Moonvest 1.2.2
macOS Apple Silicon（M1 / M2 / M3 / M4 系列）分享版

安装步骤

1. 双击 DMG，把“Moonvest”拖到“Applications”文件夹。
2. 打开 App，在“连接与风控”中填写最多两个 Moonvest follow 用户，并把自己的 API key 保存到 macOS 登录钥匙串。
3. 使用 moomoo 时只需先启动 OpenD；切换券商后无需先保存即可发现账户。使用 Robinhood 时点击“连接 Robinhood”，在官方页面完成 OAuth 授权，再返回 App 选择账户并一次性保存。
4. 新安装默认是模拟盘 + 仅观察，不会提交订单。

macOS 首次打开提示

此分享版经过本地 ad-hoc 签名，但没有 Apple Developer ID 公证。如果 macOS 阻止首次启动：

1. 在 Finder 的“应用程序”中按住 Control 点击“Moonvest”；
2. 选择“打开”；
3. 在确认框中再次选择“打开”。

也可以进入“系统设置 → 隐私与安全性”，在安全提示旁点击“仍要打开”。

安全说明

- Moonvest API key 只保存到接收者自己的 macOS 登录钥匙串，不进入设置文件、数据库、日志或分享包。
- 分享版使用 App 内置可信 CA 验证 Moonvest 与券商 HTTPS，不依赖接收者安装 Python 证书。
- App 不接收券商登录密码或交易密码。Robinhood 连接全程使用官方 OAuth 页面；Webull / 嘉信 API 密钥与令牌同样只保存到登录钥匙串。
- 唯一开单信息源是 Moonvest Active Trades SSE；没有手动单、Webhook 或其他来源入口。
- 事件 id 用于持久去重、cursor 恢复和下游幂等标识。
- SSE 断线或 resync 会自动关闭订单执行；连接恢复不会自动重新启用下单。
- “启用执行”是独立的临时开关，4 小时后自动关闭。
- moomoo OpenD 支持股票与单腿期权执行。Robinhood 通过官方 Trading MCP 支持美股/ETF 执行，并强制先预检再下单；普通 Robinhood 账户只读，只有独立 Agentic 账户可执行。IBKR、Webull 与嘉信仍为只读接入。
- moomoo 实盘必须先在 OpenD GUI 中由使用者手动解锁。
- 所有个人配置与审计数据只保存在接收者自己的：
  ~/Library/Application Support/Moonvest/

系统要求

- macOS 12.0 或更高版本
- Apple Silicon Mac（ARM64）
- 使用 moomoo 时需安装并运行 OpenD
- 使用 Robinhood 时需有状态正常的 Robinhood 个人账户，并在桌面浏览器完成 Agentic 账户授权
- 接收 Moonvest SSE 时需要联网
